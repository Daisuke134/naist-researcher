# Phase 3: peer-review プロンプト（JSPS DC1評価基準）

## タスク

採択例2本を参照基準として、生成した研究計画書をスコアリングし、改善フィードバックを生成する。

## 評価対象

- **レビュー対象**: `writing_outputs/[timestamp]_dc1/drafts/v{N}_draft.md`
- **採択基準1**: `data/jsps-examples/yoshida-dc1-2011.md`（面接免除採択レベル）
- **採択基準2**: `data/jsps-examples/tanisumi-dc1-2019.md`（採択レベル）

## 採点基準（合計100点）

| # | 評価項目 | 配点 | 採点基準 |
|---|---------|------|---------|
| 1 | 社会的背景 | 20点 | 冒頭に数値（罹患率・患者数・社会コスト）があれば20点。数値なし・曖昧なら10点以下 |
| 2 | ファネル構造 | 15点 | 社会課題→技術課題→本研究への段階的絞り込みが明確なら15点 |
| 3 | 新規性の明確さ | 20点 | 「〜は未解決」→「本研究では〜で解明」の2段構造が明確なら20点 |
| 4 | 研究方法の具体性 | 20点 | 年度別計画・使用手法・評価指標が全て明記されていれば20点 |
| 5 | 図・概念図 | 10点 | 研究フロー図・概念図が1枚以上あれば10点 |
| 6 | 期待成果・社会貢献 | 15点 | 臨床応用・社会実装への具体的道筋があれば15点 |

## 採点方法

各項目について採択例2本と比較して採点する。

採点例:
```
【社会的背景】
- 吉田DC1: 「てんかんは100人に1人、国内100万人罹患」と明記 → 20点
- 当該申請書: 「○○は○人に1人」という数値記述がない → 8点
```

## fix_plan.md の更新ルール（スコア < 85 の場合）

改善箇所を `.ralph/fix_plan.md` に以下の形式で追記:

```markdown
## Round {N} の弱点（スコア: {SCORE}/100）
- [ ] {改善項目1} → {具体的な修正内容}
- [ ] {改善項目2} → {具体的な修正内容}
- [ ] {改善項目3} → {具体的な修正内容}

## Round {N} 完了済み
（Phase 2でチェックを入れること）
```
